# タスクリスト生成

## トリガー

「始めたい」「プロジェクトを立ち上げたい」「タスクリスト作って」「何から始めればいい？」等

## 概要

クライアントとの対話を通じてプロジェクトの全体像を把握し、5〜10項目のタスクリストを `TASKLIST.md` として生成する。

## 手順

### Step 1: プロジェクト概要のヒアリング

以下の3点を順番に聞く（一度に全部聞かない）:

**まず最初に:**
「どんなものを作りたいですか？ざっくりで大丈夫です。」

**回答を受けて:**
「なるほど。それを作りたい背景や目的を教えてください。誰のために、何を解決したいですか？」

**さらに:**
「何か制約はありますか？（例: 予算、期限、使いたい技術、避けたいことなど）」

### Step 2: プロジェクト種別の判定

ヒアリング内容から、プロジェクトの種別を判定する:

| 種別 | 判定基準 |
|---|---|
| Webサービス開発 | ユーザー向けのWebアプリ、サービス、サイト |
| 社内ツール | 業務効率化、社内システム、管理ツール |
| AI活用・導入 | AI機能の組み込み、ChatGPT活用、自動化 |
| その他 | 上記に当てはまらない |

判定したら、対応するテンプレートを参考にする:
- Webサービス開発 → `templates/web-service.md`
- 社内ツール → `templates/internal-tool.md`
- AI活用・導入 → `templates/ai-integration.md`
- その他 → テンプレートなしで、ヒアリング内容から直接生成

### Step 3: タスクリストのカスタマイズ

テンプレートをベースに、ヒアリング内容に合わせてカスタマイズする:

1. テンプレートのタスクで**不要なものを削除**
2. ヒアリングで出た**固有の要件をタスクとして追加**
3. **佐藤のレビューが必要そうなポイント**があれば「佐藤に確認」タスクを挿入
4. 最終的に **5〜10項目** に収める

### Step 4: クライアントに確認

生成したタスクリストをチャットに表示して確認してもらう:

「以下のタスクリストでいいか確認してください。追加・削除・順番の変更があれば教えてください。」

```
# プロジェクト: [プロジェクト名]

## タスクリスト

- [ ] タスク1: ...
- [ ] タスク2: ...
- [ ] タスク3: ...
...
```

### Step 5: TASKLIST.md の生成

クライアントが OK を出したら、ワークスペースルートに `TASKLIST.md` を作成する。

ファイルの形式:

```markdown
# プロジェクト: [プロジェクト名]

作成日: [YYYY-MM-DD]

## 概要

[ヒアリングで得たプロジェクト概要を2〜3行で]

## タスクリスト

- [ ] タスク1: [内容]
- [ ] タスク2: [内容]
- [ ] タスク3: [内容]
...
```

生成後:
「タスクリストを作成しました！最初のタスク『[タスク1の名前]』から始めましょうか？」

## 注意事項

- タスクの粒度は「クライアントが1回の作業セッションで進められる」くらいを目安にする
- 技術的すぎるタスク名にしない（「DB設計」ではなく「データの保存方法を決める」など）
- テンプレートはあくまで参考。クライアントの要望を最優先する
- タスクリストは後から変更できることを伝えて安心させる
