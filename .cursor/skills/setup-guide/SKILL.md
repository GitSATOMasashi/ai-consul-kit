# セットアップガイド

## トリガー

「セットアップしたい」「Cursorの使い方がわからない」「環境構築」「初期設定」等

## 前提

- クライアントは Mac 環境
- 佐藤がオンラインミーティング（Google Meet）で画面共有しながらサポートする想定
- このスキルは対話形式で進める。一度に全部説明せず、ステップごとに確認を挟む

## 手順

### Step 1: 現在の状況確認

クライアントに以下を質問して、どこから始めるか判断する:

1. 「Cursor はもうインストールしていますか？」
2. 「GitHub のアカウントは持っていますか？」

回答に応じて、必要なステップだけを実行する。

### Step 2: Cursor のインストールと初期設定

**Cursor 未インストールの場合:**

以下を案内する:

1. 「https://cursor.com にアクセスして、Cursor をダウンロードしてください」
2. 「ダウンロードしたファイルを開いて、アプリケーションフォルダにドラッグしてください」
3. 「Cursor を起動してください。初回起動時にアカウント作成を求められるので、メールアドレスで登録してください」

**Cursor インストール済みの場合:**

日本語化と推奨設定を案内する:

1. 日本語化:
   - 「`Cmd + Shift + P` を押してください（コマンドパレットが開きます）」
   - 「`Configure Display Language` と入力して、表示された候補をクリックしてください」
   - 「`ja` を選択してください。Cursor が再起動して日本語になります」

2. 推奨設定:
   - 「左下の歯車アイコン → 設定 を開いてください」
   - 「Auto Save を `afterDelay` に変更しましょう（ファイルが自動保存されるようになります）」

設定が終わったら「ここまで大丈夫ですか？」と確認する。

### Step 3: GitHub アカウントとリポジトリのクローン

**GitHub アカウント未作成の場合:**

1. 「https://github.com にアクセスして、Sign up からアカウントを作りましょう」
2. 「メールアドレス、パスワード、ユーザー名を設定してください」
3. 「メールに届く確認コードを入力して登録完了です」

**リポジトリのクローン:**

1. 「Cursor の左上メニューから『ターミナル → 新しいターミナル』を選んでください」
2. 「下にターミナル（黒い画面）が出てきたら、以下をそのままコピー＆ペーストしてください:」
   ```
   cd ~/Desktop && git clone [リポジトリURL]
   ```
   ※ リポジトリURLは佐藤が事前に共有する
3. 「Cursor のメニューから『ファイル → フォルダーを開く』で、今クローンしたフォルダを開いてください」

### Step 4: Cursor の基本操作ガイド

以下を実際に操作しながら案内する:

1. **チャットの使い方**:
   - 「右サイドバーにチャット欄があります。ここに話しかけるとAIが答えてくれます」
   - 「試しに『こんにちは』と送ってみてください」

2. **Agentモードの確認**:
   - 「チャット欄の上部に『Agent』と表示されていれば OK です」
   - 「Agent モードでは、AIがファイルを作ったり編集したりできます」

3. **ファイル参照の使い方**:
   - 「チャットで `@` をつけるとファイルを指定できます。後で使うかもしれないので覚えておいてください」

### Step 5: 完了確認

以下のチェックリストを使って、セットアップが完了しているか確認する:

- [ ] Cursor がインストールされ、起動できる
- [ ] Cursor が日本語表示になっている
- [ ] GitHub アカウントがある
- [ ] リポジトリがクローンされ、Cursor で開けている
- [ ] チャットでAIと会話できる

すべて OK なら:
「セットアップ完了です！次のステップに進みましょう。」

## 注意事項

- ターミナル操作は最小限にする。コピペだけで済むように案内する
- エラーが出た場合は「画面をスクリーンショットで送ってください」と案内する
- SSH キーの設定は初回では不要（HTTPS クローンで十分）
- 佐藤の画面共有サポートが前提なので、テキストだけで完結しなくても問題ない
